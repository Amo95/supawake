A tiny CLI that pings your free-tier Supabase databases on a schedule so they never auto-pause after 7 days of inactivity.

View on GitHub How it works
npm install -g supawake

The problem

Supabase pauses free-tier projects after roughly 7 days of inactivity. Your side project, demo, or portfolio app goes down, and users see errors. Unpausing requires logging into the dashboard manually and waiting for the database to restart.

supawake fixes this with a single scheduled HTTP request. It hits the Supabase REST API using your API key, which counts as activity and keeps your project alive — forever.

Features

⏱

Scheduled pinging

Run continuously with a cron schedule, or one-shot from CI. Default: every 3 days.

📊

Multi-project

Manage unlimited Supabase projects from a single config file.

🔔

Failure alerts

Optional webhook notifications (Slack, Discord, ntfy.sh) when a ping fails.

⚙

GitHub Actions ready

Includes a workflow template. Let GitHub run your pings for free — zero infra needed.

How it works

1

Add your projects

Run supawake add and enter your project URL and API key.

2

Verify connectivity

Run supawake status to confirm your projects are reachable.

3

Schedule pings

Use supawake start locally, or set up the GitHub Action to run it for free.

# Add a project
supawake add

# Check status
supawake status

# Ping all projects once
supawake ping

# Run on a schedule (every 3 days)
supawake start

🚀 Recommended: run with GitHub Actions

Create a private repo with one workflow file, store your config as a secret, and GitHub will ping your projects every 3 days for free. No server, no Docker, no cron on your laptop. See setup instructions →

CLI reference

supawake add              # Add a Supabase project (interactive)
supawake remove <name>   # Remove a project by name
supawake list             # List all configured projects
supawake ping             # Ping all projects once
supawake start            # Continuous pinging on a schedule
supawake status           # Check which projects are reachable
